# 多用户系统

TeamClaw v3.0 引入了完整的企业级多用户认证体系，从个人工具正式升级为团队协作平台。

## 三种角色

| 角色 | 权限范围 |
|------|----------|
| **Admin** | 全部权限：系统配置、用户管理、审批、所有数据 |
| **Member** | 参与项目协作：任务、Wiki、交付、聊天 |
| **Viewer** | 只读权限：浏览项目内容，不能操作 |

## 安全码二次验证

敏感操作需要二次验证：
- 用户密码修改
- Gateway 配置变更
- Skill 安装 / 卸载
- 项目删除

## 用户隔离

多用户模式下，数据完全隔离：
- 聊天会话按用户分离
- 个人配置独立存储
- 项目数据按成员权限访问

## 用户注册

支持两种注册模式：
- **开放注册**：任何人可注册（适合内部团队）
- **邀请制**：仅管理员可邀请新成员

## 部署要求

多用户系统需要：
- 首次部署时创建管理员账户
- 配置 Gateway 连接（WebSocket 直连模式）
- 建议配合反向代理和 HTTPS 使用
