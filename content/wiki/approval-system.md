# 通用审批系统

TeamClaw 的统一审批引擎，为所有敏感操作提供规范化的审核流程。

## 4 种审批类型

| 类型 | 说明 |
|------|------|
| `skill_publish` | Skill 发布审批 |
| `skill_install` | Skill 安装审批 |
| `project_join` | 项目加入申请 |
| `sensitive_action` | 敏感操作审批（预留） |

## 审批流程

1. **申请人发起请求** — 填写申请表单
2. **系统路由到审批人** — 根据配置的审批策略
3. **审批人审核** — 查看详情，决定通过/拒绝
4. **申请人收到通知** — 通过则自动执行，拒绝则告知原因

## 审批策略配置

| 策略 | 说明 |
|------|------|
| `any_admin` | 任意管理员审批即可 |
| `specific_role` | 指定角色审批 |
| `project_admin` | 项目管理员审批 |

## 超时自动处理

可配置超时时间，到期后自动处理：
- **自动批准**
- **自动拒绝**
- **保持待处理**（不自动处理）

## 权限矩阵

| 操作 | 申请人 | 管理员 | 其他用户 |
|------|--------|--------|----------|
| 创建审批请求 | ✅ | ✅ | ✅ |
| 查看自己发出的请求 | ✅ | ✅ | ✅ |
| 查看所有请求 | ❌ | ✅ | ❌ |
| 批准 / 拒绝 | ❌ | ✅ | ❌ |
| 取消自己的请求 | ✅ | ✅ | ❌ |

## 通知配置

支持为申请人和审批人配置通知：
- 申请提交时通知审批人
- 审批完成时通知申请人
