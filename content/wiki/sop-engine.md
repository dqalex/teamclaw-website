# SOP 工作流引擎

SOP（Standard Operating Procedure）工作流引擎是 TeamClaw 的核心能力之一。让 AI Agent 按预定义的标准化流程执行复杂任务，而不是靠模糊的指令随机发挥。

## 为什么需要 SOP

没有 SOP 的 AI：输出质量不稳定、步骤遗漏、上下文混乱。

有 SOP 的 AI：每步都有明确产出、人工在关键节点把关、知识自动沉淀。

## 7 种阶段类型

| 阶段类型 | 说明 | 适用场景 |
|----------|------|----------|
| `input` | 人工输入阶段，收集表单数据 | 需求收集、参数配置 |
| `ai_auto` | AI 自动执行，完成后自动推进 | 内容生成、代码编写 |
| `ai_with_confirm` | AI 执行后需人工确认才推进 | 敏感操作、关键决策 |
| `manual` | 纯人工操作阶段 | 需要人工介入的环节 |
| `render` | 可视化渲染，自动创建文档 | 报告生成、周报月报 |
| `export` | 导出阶段，输出到外部平台 | 发布到腾讯文档、飞书等 |
| `review` | 最终审核阶段 | 质量把控、最终确认 |

## 内置 SOP 模板

TeamClaw 提供了 5 个开箱即用的模板：

### 竞品调研 SOP（5 阶段）
`需求确认 → AI 调研 → 报告生成 → 审核 → 发布`

### 内容营销 SOP（6 阶段）
`选题 → 创作 → 审核 → 排版 → 发布 → 数据追踪`

### 周报月报 SOP（4 阶段）
`数据收集 → AI 汇总 → 渲染生成 → 审核确认`

### Bug 分析 SOP（5 阶段）
`问题复现 → 根因分析 → 修复方案 → 验证 → 知识沉淀`

### 数据分析 SOP（5 阶段）
`需求定义 → 数据采集 → 分析处理 → 可视化 → 审核发布`

## L1-L5 知识库分层

每个 SOP 模板可关联 Know-how 文档库，按五层结构组织：

- **L1 概述** — 背景和目标
- **L2 核心知识** — 关键方法论和分析框架
- **L3 工具方法** — 具体工具的使用方法
- **L4 经验教训** — 历史执行中的经验，AI 自动追加
- **L5 统计数据** — 执行成功率、平均耗时等

## 如何使用

1. 打开任意任务
2. 点击「关联 SOP」按钮
3. 从模板库选择或新建 SOP
4. AI 自动按阶段执行，人工节点需确认推进
5. 完成所有阶段后自动交付

## SOP 设计原则

- **阶段粒度适中**：每个阶段应有明确产出物
- **人机结合**：AI 擅长的用 `ai_auto`，需要判断的用 `ai_with_confirm`
- **知识沉淀**：每个 SOP 都关联 Know-how 文档，持续积累
- **持续优化**：基于 L4 经验数据迭代 SOP 模板
